# 코딩 테스트 문제 - 2025.12.31

## 문제: 회의실 배정

### 문제 설명
한 개의 회의실이 있는데, 이를 사용하고자 하는 N개의 회의에 대하여 회의실 사용표를 만들려고 합니다.
각 회의는 시작시간과 끝나는 시간이 주어져 있고, 각 회의가 겹치지 않게 하면서 회의실을 사용할 수 있는 회의의 최대 개수를 구하는 프로그램을 작성하세요.

단, 회의는 한번 시작하면 중간에 중단될 수 없으며, 한 회의가 끝나는 것과 동시에 다음 회의가 시작될 수 있습니다. (즉, 회의의 시작 시간과 끝나는 시간이 같을 수도 있습니다. 이 경우에는 겹치지 않는 것으로 간주합니다.)

### 입력
- 첫째 줄에 회의의 수 N(1 ≤ N ≤ 100,000)이 주어집니다.
- 둘째 줄부터 N+1 줄까지 각 회의의 정보가 주어집니다.
- 각 줄에는 회의의 시작시간과 끝나는 시간이 공백으로 구분되어 주어집니다. (0 ≤ 시작시간 < 끝나는 시간 ≤ 2^31-1)

### 출력
첫째 줄에 최대 사용할 수 있는 회의의 개수를 출력합니다.

### 입력 예제 1
```
11
1 4
3 5
0 6
5 7
3 8
5 9
6 10
8 11
8 12
2 13
12 14
```

### 출력 예제 1
```
4
```

### 입력 예제 2
```
5
1 3
2 4
3 5
4 6
5 7
```

### 출력 예제 2
```
4
```

### 제약 사항
- 시간 제한: 2초
- 메모리 제한: 128MB

### 힌트
그리디(Greedy) 알고리즘을 고려해보세요.
