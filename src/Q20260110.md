# Q20260110 - 구명보트 🚤

## 📋 문제 설명
무인도에 갇힌 사람들을 구명보트를 이용하여 구출하려고 합니다. 구명보트는 작아서 한 번에 **최대 2명**씩 밖에 탈 수 없고, 무게 제한도 있습니다.

예를 들어, 사람들의 몸무게가 [70kg, 50kg, 80kg, 50kg]이고 구명보트의 무게 제한이 100kg이라면 2번째 사람과 4번째 사람은 같이 탈 수 있지만 1번째 사람과 4번째 사람의 무게의 합은 120kg이므로 같이 탈 수 없습니다.

구명보트를 최대한 적게 사용하여 모든 사람을 구출하려고 합니다.

사람들의 몸무게를 담은 배열 `people`과 구명보트의 무게 제한 `limit`가 매개변수로 주어질 때, **모든 사람을 구출하기 위해 필요한 구명보트 개수의 최솟값**을 return 하도록 solution 함수를 작성해주세요.

## 📥 입력 형식
- `people`은 사람들의 몸무게를 담은 배열이며, 크기는 1 이상 50,000 이하입니다.
- `people`의 각 원소는 40 이상 240 이하의 자연수입니다.
- `limit`는 구명보트의 무게 제한이며, 40 이상 240 이하의 자연수입니다.
- 구명보트는 항상 모든 사람의 몸무게보다 크거나 같습니다.

## 📤 출력 형식
- 모든 사람을 구출하기 위해 필요한 구명보트 개수의 최솟값을 return 합니다.

## 💡 입출력 예

| people | limit | return |
|--------|-------|--------|
| [70, 50, 80, 50] | 100 | 3 |
| [70, 80, 50] | 100 | 3 |
| [40, 40, 40, 60, 80] | 120 | 3 |
| [100, 500, 500, 900] | 1000 | 2 |

### 입출력 예 설명

**예제 #1**
- 70kg + 50kg = 120kg > 100kg (불가)
- 80kg (혼자)
- 50kg + 50kg = 100kg (가능)
- 70kg (혼자)
- 총 3개의 보트 필요

**예제 #2**
- 각자 혼자 타야 함
- 총 3개의 보트 필요

**예제 #3**
- 40kg + 80kg = 120kg (가능)
- 40kg + 60kg = 100kg (가능)  
- 40kg (혼자)
- 총 3개의 보트 필요

**예제 #4**
- 정렬: [100, 500, 500, 900]
- 1번 보트: 100kg + 900kg = 1000kg (2명 함께 가능) ✅
- 2번 보트: 500kg + 500kg = 1000kg (2명 함께 가능) ✅
- 총 2개의 보트 필요

```
while (left <= right) {
    if (people[left] + people[right] <= limit) {
        // 두 명 함께 탑승
        left++;
        right--;
    } else {
        // 무거운 사람만 탑승
        right--;
    }
    boats++;
}
```

<details>
<summary>힌트 3: 왜 이 방법이 최적일까요?</summary>

**그리디의 정당성**:
- 가장 무거운 사람은 결국 보트를 타야 함
- 이 사람과 함께 탈 수 있는 사람은 가장 가벼운 사람뿐
- 만약 가장 가벼운 사람도 함께 못 탄다면, 다른 누구와도 함께 탈 수 없음
- 따라서 매 순간 "가장 무거운 + 가장 가벼운" 매칭이 최적!
